01 / MOTIVATION

Humanoid learning needs more than a policy checkpoint.

A useful humanoid manipulation system must connect how a person demonstrates a task, how robot trajectories are represented, how a policy is trained, and how failure is contained before hardware execution. This project treats those parts as one engineering loop.

HUMAN INPUT

Two ways to demonstrate

Upper-body manipulation benefits from precise XR control, while full-body behavior requires coordinated whole-body motion.

DATA ENGINE

One learnable representation

Heterogeneous demonstrations must become synchronized, inspectable LeRobot datasets before they can support repeatable training.

SAFE ITERATION

Simulation before hardware

Policy behavior is checked in simulation before commands are allowed to reach the physical G1.

03 / TELEOPERATION

Capture intent at the right level of embodiment.

The project uses two complementary systems rather than forcing every demonstration through one controller.

SYSTEM A / UPPER BODY

Apple Vision Pro teleoperation

An XR-based interface for commanding the G1 upper body and Inspire hands, designed for close-range bimanual manipulation and precise human demonstrations.

  • Upper-body and hand commands
  • First-person visual feedback
  • Manipulation-focused demonstrations
SYSTEM B / WHOLE BODY

Whole-body TWIST2 reproduction

A full-system reproduction of the TWIST2 pipeline for mapping coordinated human motion to the complete G1 body, extending demonstrations beyond tabletop arm motion.

  • Whole-body motion mapping
  • Coordinated upper and lower body
  • Portable demonstration collection
04 / DATA

05 / POLICY LEARNING

Fine-tune π0.5 for the G1 action space.

The standardized demonstrations are used with the unitree_lerobot training stack to train a π0.5 policy through imitation learning.

INPUT

Vision + language + robot state

Camera observations, task instruction, and the robot's proprioceptive state form the policy context.

POLICYπ0.5IMITATION LEARNING
OUTPUT

G1 manipulation actions

The trained policy predicts action commands aligned with the converted G1 state-action representation.

06 / VALIDATION

Verify in simulation. Then earn the right to move hardware.

Validation is staged: policy inference and interfaces are checked in Isaac Sim before the same learned behavior is deployed to the physical G1 for real-world evaluation.

STAGE 01 / SIMULATION

Isaac Sim

Check observation pipelines, action mapping, timing, task rollout, and obvious failure modes without exposing the physical robot to the first iteration of a policy.

STAGE 02 / REAL ROBOT

Physical G1 deployment

Deploy the trained model to the physical G1, run closed-loop inference with live observations, and evaluate whether the learned behavior survives the sim-to-real gap.
